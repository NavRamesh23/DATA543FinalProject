What this dataset is: Every CVE here was not just discovered — it was actively used by real attackers. CISA adds a CVE to KEV only after confirming real exploitation. This makes KEV the highest-signal threat dataset for defenders.

What it is NOT: This is not all CVEs. There are 200,000+ total CVEs ever published; KEV contains ~1,539 (~0.8%). All analysis here reflects real-world attacker behavior, not theoretical risk.
⚠️ "For one year" caveat: When filtering to one year (e.g., 2024), you are seeing CVEs that CISA confirmed were being exploited that year. The actual CVE may have been published years earlier. Always state: "These are KEV CVEs confirmed exploited in [year], not all CVEs published that year."

Credential Access — Deep Dive
Is Credential Access (TA0006) an inside or outside tactic? Where in the attack chain does it appear?
Answer: Always inside. Credential Access requires the attacker to already have some presence in the system. They cannot dump passwords from memory (LSASS), steal Kerberos tickets, or read credential stores without first getting in.

The attack chain for credential theft:
1️⃣ Initial Access — attacker gets in (via exploited CVE, phishing, etc.)
2️⃣ Execution — attacker runs code to establish a foothold
3️⃣ Credential Access — attacker steals real usernames/passwords/tokens
4️⃣ Now they look like a legitimate user — detection becomes very hard
5️⃣ Lateral Movement / Impact — they move through the network or cause damage
Why credential theft is so dangerous: Once an attacker has valid credentials, their activity blends in with normal user behavior. Security tools that look for "unusual" behavior may miss this completely.
